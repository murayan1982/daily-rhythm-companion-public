# Daily Rhythm Companion v1.5.0 Release Notes

## Release summary

Daily Rhythm Companion v1.5.0 is a Mood and personalization foundation release.

It makes the daily advice loop feel more personal by adding character-aware mood choice copy and test coverage while keeping the app-facing mood IDs stable and preserving safe, conservative health wording.

Fixed release zip:

```text
release\DailyRhythmCompanion_20260521_221101.zip
```

Release positioning:

```text
Mood and personalization foundation for the AI Character Framework demo app.
```

This release does not rebuild the fixed zip after release notes are written. `release_notes/v1.5.0.md` is the source-tree release record for the fixed zip above.

## Mood and personalization policy

v1.5.0 keeps the canonical mood IDs stable:

```text
energetic
normal
tired
```

Character-aware mood labels are presentation-only. They make the Flutter mood picker feel more character-aware without changing the backend API contract, advice request mood value, or DailyRecord mood storage.

Current character-aware mood labels:

```text

  energetic -> いい感じ
  normal    -> いつも通り
  tired     -> ちょっと休みたい

cheerful_sora:
  energetic -> いけそう！
  normal    -> ぼちぼち
  tired     -> 省エネで

  energetic -> 高め
  normal    -> 標準
  tired     -> 低め
```

The Flutter widget test confirms that the label can be character-specific while the advice request still uses stable mood IDs.

## Lightweight personalization boundary

v1.5.0 defines lightweight personalization candidates without adding persistent user-profile infrastructure yet.

Allowed future candidates:

```text
nickname
preferred_mood_labels
advice_focus_preference
tone_preference
```

Explicitly out of scope for v1.5.0:

```text
profile persistence
backend user-profile endpoints
AdviceRequest schema changes
DailyRecord schema changes
provider memory
medical profiling / diagnosis
```

This keeps personalization lightweight, local to the app/demo surface, and safe for mock-safe operation.

## Safety and default behavior

v1.5.0 keeps the same safe-default policy:

```text
- mock-safe mode works without external credentials.
- normal source-tree checks do not require a real AI Character Framework checkout.
- mood personalization does not require provider-backed LLM execution.
- release checks do not call external LLM providers.
- Google Health real API execution remains explicit opt-in only.
- release zip verification uses a fixed artifact and does not rebuild it.
- mood labels are presentation-only and do not change the stable mood IDs.
```

---

## v1.5.0 does not claim

```text
- production hosted consumer-service readiness
- App Store / Google Play distribution
- provider-backed LLM execution as a requirement
- mandatory AI Character Framework checkout for normal checks
- automatic real health-data access without explicit opt-in
- medical diagnosis, treatment advice, or health improvement guarantees
- full personalization/profile platform readiness
- long-term provider memory or cloud profile storage
```
