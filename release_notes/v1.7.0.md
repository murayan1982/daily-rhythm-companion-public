# Daily Rhythm Companion v1.7.0 Release Notes

## Release summary

Daily Rhythm Companion v1.7.0 is a Rhythm report polish and app-side explanation hardening release.

It improves the app-side explanation quality for the weekly/monthly rhythm report path added in v1.6.0. The History screen now presents report range, record coverage, data source, aggregation scope, and data-quality states in more user-facing wording while preserving conservative, non-medical wording and the mock-safe default.

Fixed release zip:

```text
release\DailyRhythmCompanion_20260522_214532.zip
```

Release positioning:

```text
Rhythm report polish and app-side explanation hardening for the AI Character Framework demo app.
```

This release does not rebuild the fixed zip after release notes are written. `release_notes/v1.7.0.md` is the source-tree release record for the fixed zip above.

## User-facing rhythm report copy

The History screen rhythm report cards now favor app-facing explanations over raw/debug-style payload labels.

Examples of the polished display direction:

```text
記録状態: 参考にしやすい記録数です
レポート範囲: 2026-05-11 〜 2026-05-17
保存記録: 3件 / 睡眠つき記録: 3件
データ元: 保存済みDailyRecord（デモ睡眠データ）
集計範囲: 週次の保存履歴
```

The report cards intentionally avoid making raw payload labels the main user-facing copy:

```text
Quality: usable
Source: saved_daily_record_history_with_mock_sleep
Scope: weekly_history
weekly_history
monthly_history
```

Those values may still exist in the data contract, but the app-facing display should translate them into understandable wording.

## Release cleanup and fixed-zip policy

v1.7.0 keeps the fixed-zip release workflow:

```text
Build the release zip once, record its path, then run final checks against that fixed zip without rebuilding.
```

For v1.7.0, the fixed zip was created after Day8 cleanup verification and reused for Day9 through Day12 checks.

The final fixed release zip is:

```text
release\DailyRhythmCompanion_20260522_214532.zip
```

Day12 does not create a release zip.
Day12 does not rebuild the provided zip.
Day12 verifies the provided zip as-is through the Day11 and Day10 check chain.
Day12 does not timestamp-refresh release artifacts.

## Safety and default behavior

v1.7.0 keeps the same safe-default policy:

```text
- mock-safe mode works without external credentials.
- normal source-tree checks do not require a real AI Character Framework checkout.
- rhythm report polish does not require provider-backed LLM execution.
- release checks do not call external LLM providers.
- Google Health real API execution remains explicit opt-in only.
- release zip verification uses a fixed artifact and does not rebuild it.
- weekly/monthly reports are lightweight reflections over available history.
- source and data-quality explanations remain visible and conservative.
```

---

## v1.7.0 does not claim

```text
- production hosted consumer-service readiness
- App Store / Google Play distribution
- provider-backed LLM execution as a requirement
- mandatory AI Character Framework checkout for normal checks
- automatic real health-data access without explicit opt-in
- medical diagnosis, treatment advice, or health improvement guarantees
- clinical sleep analysis or disease-risk assessment
- long-term provider memory or cloud profile storage
- complete analytics platform readiness
```
