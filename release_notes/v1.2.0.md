# Daily Rhythm Companion v1.2.0

## Release summary

Daily Rhythm Companion v1.2.0 is a Google Health real-use onboarding readiness release.

It improves the project’s Google Health configured local/demo path by adding clearer setup, recovery, configured verification, wording, non-exposure, package, and final release checks.

Fixed release zip:



## Safety and guard policy

v1.2.0 keeps the existing safe-default policy:

```text
Safe default + documented explicit opt-in.
```

Mock-safe behavior remains the normal default:

```text
- no Google credentials required
- no accidental real API calls
- missing optional Google configuration should not break normal local checks
- configured-only checks skip safely unless explicit opt-in gates are enabled
```

Configured real/demo verification requires explicit opt-in gates such as:

```text
GOOGLE_HEALTH_ENABLE_REAL_TOKEN_EXCHANGE=1
GOOGLE_HEALTH_ENABLE_REAL_TOKEN_REFRESH=1
GOOGLE_HEALTH_ENABLE_REAL_API_REQUESTS=1
GOOGLE_HEALTH_REAL_API_OPT_IN=1
GOOGLE_HEALTH_REAL_ENDPOINT_VERIFIED=1
```

The committed mock-safe profile keeps those gates disabled.

## Google Health wording and non-exposure

v1.2.0 standardizes safe Google Health state wording:

```text
not_configured
authorization_required
connected
reconnect_required
unavailable
fallback_to_mock
configured_skip
```

It also reinforces source labels:

```text
mock
google_health
fallback
unavailable
```

The release checks protect against exposing sensitive-looking values in Google Health docs/checks and release packages, including:

```text
- access tokens
- refresh tokens
- client secrets
- authorization codes
- API keys
- raw Google Health payloads
- local token files
- private absolute paths
```
