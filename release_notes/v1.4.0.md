# Daily Rhythm Companion v1.4.0 Release Notes

Daily Rhythm Companion v1.4.0 is a Character experience expansion release.

It makes the bundled demo characters easier to understand and compare while keeping the app-facing character contract small, explicit, and safe for mock-safe local development.

## Character experience policy

v1.4.0 keeps the bundled character surface intentionally small.

Current bundled characters:

```text
gentle_mina / ミナ
cheerful_sora / ソラ
cool_rei / レイ
```

The release separates stable app-facing character contract fields from presentation and tone guidance.

Stable app-facing fields include:

```text
character_id
display_name
description
personality_type
speaking_style
advice_style
```

Selection-facing copy and advice tone guidance are documented separately so the demo can show character differences without turning DRC into a large character authoring platform.

---

## Framework mapping policy

v1.4.0 documents the DRC character_id to AI Character Framework mapping contract.

Current bundled characters intentionally map to the framework `default` character unless a configured local/demo operator overrides `FRAMEWORK_CHARACTER`:

```text
gentle_mina -> default
cheerful_sora -> default
cool_rei -> default
```

This keeps mock-safe local development independent from a real framework checkout or provider-backed LLM credentials.

## Safety and default behavior

v1.4.0 keeps the same safe-default policy:

```text
- mock-safe mode works without external credentials.
- normal source-tree checks do not require a real AI Character Framework checkout.
- character expansion does not require provider-backed LLM execution.
- release checks do not call external LLM providers.
- Google Health real API execution remains explicit opt-in only.
- release zip verification uses a fixed artifact and does not rebuild it.
```

---

## v1.4.0 does not claim

```text
- production hosted consumer-service readiness
- App Store / Google Play distribution
- mandatory provider-backed LLM execution
- mandatory AI Character Framework checkout for normal checks
- automatic real health-data access without explicit opt-in
- medical diagnosis, treatment advice, or health improvement guarantees
- full character authoring platform readiness
```
