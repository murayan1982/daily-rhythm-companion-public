# Daily Rhythm Companion v1.8.0 Release Notes

## Release summary

Daily Rhythm Companion v1.8.0 is a Report-to-advice handoff and DailyRecord reflection polish release.

It connects the weekly/monthly rhythm report surface from v1.6.0 and the app-side report explanation polish from v1.7.0 to the daily advice and saved-history loop. The release adds a small, explicit `ReportHandoffContext` boundary, optional advice metadata, Flutter display helpers, and History reflection copy so report-informed advice can be shown without presenting historical reports as today's sleep or as medical analysis.

Fixed release zip:

```text
release\DailyRhythmCompanion_20260522_234744.zip
```

Release positioning:

```text
Report-to-advice handoff and DailyRecord reflection polish for the AI Character Framework demo app.
```

This release does not rebuild the fixed zip after release notes are written. `release_notes/v1.8.0.md` is the source-tree release record for the fixed zip above.

## User-facing report-informed advice and reflection

The app can now show that advice or a saved DailyRecord used historical report context without exposing raw/debug-style labels as the main user-facing copy.

The intended display direction is:

```text
最近のリズムレポートも参考にしています
過去の保存記録をもとにした補助的な文脈です
```

The UI and reflection copy should continue to avoid making raw payload labels the primary user-facing wording:

```text
source_label
data_scope
data_quality
rhythm_report+mood+character+mock
rhythm_report_partial+mood+character+framework_fallback
```

Those values may remain in the data contract and debug-friendly metadata, but app-facing wording should translate them into understandable, conservative explanations.

## Backend behavior

Updated backend files include:

```text
backend/app/models/report_handoff.py
backend/app/services/report_handoff_service.py
backend/app/models/advice.py
backend/app/api/advice.py
backend/app/services/advice_prompt_builder.py
```

The backend now has a small report handoff boundary for:

```text
- preserving report source/data-quality/scope metadata
- deciding whether report context should inform advice
- using rhythm_report for usable report context
- using rhythm_report_partial for partial report context
- dropping insufficient or unsafe report context
- adding conservative prompt guidance through build_report_handoff_prompt_section that treats reports as historical context only
```

The handoff remains optional. Existing clients can keep calling `/advice` without `report_handoff` metadata.

## Safety and default behavior

v1.8.0 keeps the same safe-default policy:

```text
- mock-safe mode works without external credentials.
- normal source-tree checks do not require a real AI Character Framework checkout.
- report-informed advice does not require provider-backed LLM execution.
- release checks do not call external LLM providers.
- Google Health real API execution remains explicit opt-in only.
- release zip verification uses a fixed artifact and does not rebuild it.
- weekly/monthly reports are historical context over available saved records.
- report-informed advice is not presented as today's sleep data.
- source and data-quality explanations remain visible and conservative.
```

---

## v1.8.0 does not claim

```text
- production hosted consumer-service readiness
- App Store / Google Play distribution
- provider-backed LLM execution as a requirement
- mandatory AI Character Framework checkout for normal checks
- automatic real health-data access without explicit opt-in
- medical diagnosis, treatment advice, or health improvement guarantees
- clinical sleep analysis or disease-risk assessment
- long-term provider memory or cloud profile storage
- complete analytics platform readiness
- automatic report selection before advice without an explicit handoff context
```
