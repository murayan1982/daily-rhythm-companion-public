## Safety and default behavior

v1.3.0 keeps the same safe-default policy:

```text
- mock-safe mode works without external credentials.
- normal source-tree checks do not require a real AI Character Framework checkout.
- configured-only smoke checks print SKIP when FRAMEWORK_ROOT / FRAMEWORK_PROJECT_ROOT is missing.
- provider-backed LLM verification requires an explicit opt-in gate.
- release checks do not call external LLM providers.
- provider API keys must stay in private local environment variables and must not be committed.
- release zip verification uses a fixed artifact and does not rebuild it.
```

Optional provider-backed ask verification remains explicit:

```powershell
$env:DRC_V130_ENABLE_CONFIGURED_LLM_SMOKE = "1"
$env:GEMINI_API_KEY = "<provider-api-key>"
python scripts\smoke_v130_framework_llm_configured_demo.py --ask
```

## v1.3.0 does not claim

```text
- production hosted consumer-service readiness
- App Store / Google Play distribution
- mandatory provider-backed LLM execution
- mandatory AI Character Framework checkout for normal checks
- automatic real health-data access without explicit opt-in
- medical diagnosis, treatment advice, or health improvement guarantees
- long-term commitment to a specific LLM provider or TTS provider
```
