## Rhythm report contract

v1.6.0 uses a shared `RhythmReport` contract instead of introducing unrelated weekly and monthly models.

Current supported periods:

```text
weekly
monthly
```

The report contract is intentionally lightweight and reflection-oriented. It uses stable fields such as period, date range, source, record count, data quality, summary text, highlights, and cautions.

The weekly and monthly reports are based on saved history and demo-safe summary data. They are not presented as medical analysis, sleep diagnosis, treatment guidance, or guaranteed improvement advice.

## Backend and API behavior

v1.6.0 adds a backend foundation for rhythm reports:

```text
backend/app/models/rhythm_report.py
backend/app/services/rhythm_report_service.py
```

The app-facing API is exposed through the daily-record history surface:

```http
GET /daily-records/rhythm-report?period=weekly
GET /daily-records/rhythm-report?period=monthly
```

The API keeps source and data-quality labels visible so the UI can explain when a report is based on saved demo history, limited records, or fallback behavior.

## Safety and default behavior

v1.6.0 keeps the same safe-default policy:

```text
- mock-safe mode works without external credentials.
- normal source-tree checks do not require a real AI Character Framework checkout.
- rhythm reports do not require provider-backed LLM execution.
- release checks do not call external LLM providers.
- Google Health real API execution remains explicit opt-in only.
- release zip verification uses a fixed artifact and does not rebuild it.
- weekly/monthly reports are lightweight reflections over available history.
- source and data-quality labels remain visible and conservative.
```

---

## v1.6.0 does not claim

```text
- production hosted consumer-service readiness
- App Store / Google Play distribution
- provider-backed LLM execution as a requirement
- mandatory AI Character Framework checkout for normal checks
- automatic real health-data access without explicit opt-in
- medical diagnosis, treatment advice, or health improvement guarantees
- clinical sleep analysis or disease-risk assessment
- long-term provider memory or cloud profile storage
- complete analytics platform readiness
```
