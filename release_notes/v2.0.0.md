Release type: clean-history Public demo source release

## Publication boundary

This document describes the v2.0.0 release content. Its presence does not by itself mean that the GitHub Release has been published.

The final Public release is authorized only after the new Public repository source commit exists, the Public-distribution checks pass, one fixed zip is built from that committed Public source, Day82 and Day83 pass against the same unchanged zip, and the Public annotated `DRC_v2.0.0` tag targets the same Public source commit.

The earlier Private-repository candidate zip and tag are retained only as historical Private records and are not v2.0.0 Public release assets.

## Summary

Daily Rhythm Companion v2.0.0 is a Flutter and FastAPI demo application showing how a real app can integrate with AI Character Framework while keeping a credential-free mock-safe default.

The release combines daily rhythm context, character-based advice, optional framework-backed LLM interaction, optional voice input/output and motion capability status, Google Health sleep-summary integration, responsive Web image surfaces, and saved daily records.

## Main outcomes

```text
- Real LLM Web answer evidence accepted through the actual DRC backend and Web UI.
- Real TTS Web audio generation and audible playback evidence accepted.
- Real Google Health sleep-summary evidence accepted through backend and Web UI.
- Original character/background assets included and displayed in the Web UI.
- Public-safe evidence markers retained without raw private evidence.
- Mock-safe local operation remains credential-free.
- External provider and health-data execution require explicit opt-in.
- Clean-history Public repository migration prevents publication of Private Git history.
```

## Public safety boundaries

The Public source and release package do not include:

```text
API keys
OAuth access or refresh tokens
Authorization headers
raw provider payloads
raw prompts or answer bodies
raw audio
raw Google Health payloads or exact private sleep values
raw screenshots
private absolute paths
raw LAN IPs
operator_evidence files
local .env files
Private repository Git history or refs
```

Public evidence is marker-only. Raw evidence remains local and outside the Public repository and release package.

## Default and configured operation

Default local checks use mock-safe configuration and do not require credentials.

Configured framework, TTS, speech, motion, or Google Health paths are optional. They require explicit local setup and must expose unavailable, skipped, rejected, or fallback states instead of counting those states as configured success.

## Scope limits

Daily Rhythm Companion is a demonstration and personal-development project, not a medical device or production healthcare service. It does not diagnose, treat, or guarantee health outcomes.

General consumer App Store and Google Play publication, production hosting, production security certification, and unattended handling of real health credentials remain outside this source release.
